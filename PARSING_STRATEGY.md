# 진료내역 PDF 파서 구현 지시서 (고정 양식 특화)

> 이 문서는 Claude Code가 보험 고지의무 분석 웹앱의 **PDF 파싱 모듈**을 구현할 때 따르는 작업 지시서다.
> 핵심 원칙: **범용 표 추출기를 만들지 않는다.** 입력 PDF는 국민건강보험공단이 출력하는 3종의 고정 양식이며, 컬럼 구조·헤더 문구·코드 패턴이 항상 동일하다. 이 양식에만 특화된 결정론적 파서를 만든다.

---

## 0. 전체 아키텍처 맥락 (왜 이렇게 하는가)

- **앱 형태**: React + TypeScript + Vite 단일 페이지(SPA). 서버 없음. 모든 처리는 브라우저 안에서 끝난다. 의료 데이터는 절대 외부로 전송하지 않는다 (`fetch`/`XHR`로 원격 호출하는 코드를 두지 않는다).
- **LLM 미사용**: 파싱·집계·시술명 변환까지 전부 규칙 기반(결정론적)으로 처리한다. AI 추론이 개입하지 않으므로 할루시네이션이 원천적으로 발생하지 않는다.
- **PDF 파싱 라이브러리**: `pdfjs-dist` (PDF.js). 텍스트 조각의 글자 + x/y 좌표를 제공하므로 좌표 기반 표 재구성이 가능하다. 다른 표 추출 라이브러리는 쓰지 않는다(아래 "함정" 참고).
- **배포**: Vercel 정적 호스팅 (`vite build` → `dist/`).

이 모듈의 출력은 정규화된 행 배열 3종이며, 이후 집계 모듈이 이를 `(진료시작일 + 병의원명)`으로 join하여 고지의무 정리표를 만든다. **파서는 join까지는 하지 않고, 깨끗한 행 배열만 책임진다.**

---

## 1. 입력 PDF 3종 — 고정 스펙

세 문서 모두 동일인의 건강보험 진료내역이며, 페이지 상단에 출력 시각(`YYYY-M-D H:MM AM/PM`)과 문서 제목이 있고, 하단에 3줄짜리 고정 안내문구가 반복된다. 각 페이지 우하단에 `N/M` 페이지 번호가 있다. **대각선 워터마크 "본인 외 열람금지"가 깔려 있다 — 이 글자는 본문 텍스트 레이어에 섞여 들어올 수 있으므로 반드시 제거 대상이다.**

### 1-A. 기본진료정보 (`진료세부내역_*.pdf` 형태지만 제목은 "기본진료정보")
컬럼 (좌→우, 9개):
```
순번 | 진료시작일 | 병·의원&약국 | 진단과 | 입원/외래 | 주상병코드 | 주상병명 | 내원일수 | 총진료비(건강보험적용분) | 건강보험등혜택받은금액 | 내가낸의료비(진료비)
```
- 실제로는 금액이 3컬럼(총진료비 / 혜택금액 / 본인부담)으로 나뉜다. 합치면 11개 필드.
- **집계에 실제로 쓰는 필드**: `순번`, `진료시작일`, `병·의원&약국`, `입원/외래`(입원력 판정), `주상병코드`, `주상병명`. 나머지 금액·진단과·내원일수는 파싱은 하되 집계 필수는 아니다.
- 주상병코드 칸에는 `(양방)`/`(한방)` 표기가 코드와 별개 위치에 찍히거나 주상병명 앞에 붙는다(예: `(양방)무후각증`). **양/한방 구분을 반드시 분리 추출**한다.
- "해당없음" + 코드 자리에 `$` 가 오는 행이 있다(약국 조제행). 이 경우 상병코드는 없음 처리.

### 1-B. 처방조제정보 (`처방제조_*.pdf`, 제목 "처방조제정보")
컬럼 (8개):
```
순번 | 진료시작일 | 병·의원&약국 | 처방/조제 | 약품명 | 성분명 | 1회투약량 | 1일투여횟수 | 총투약일수
```
- **집계에 실제로 쓰는 필드**: `진료시작일`, `병·의원&약국`, `약품명`, `성분명`, `총투약일수`. (30일 누적 투약 룰 = 같은 상병그룹의 총투약일수 합산)
- 약품명·성분명이 매우 길어 **3~5줄로 줄바꿈**되는 셀이 흔하다(가장 깨지기 쉬운 컬럼).

### 1-C. 세부진료정보 (`조제정보_*.pdf`, 제목 "세부진료정보")
컬럼 (8개):
```
순번 | 진료시작일 | 병·의원&약국 | 진료내역 | 코드명 | 1회투약량 | 1일투여횟수 | 총투약일수
```
- **집계에 실제로 쓰는 필드**: `진료시작일`, `병·의원&약국`, `진료내역`(대분류), `코드명`(시술/검사/처치명).
- `진료내역` 예: `진찰료/초진(양방)`, `처치 및 수술/처치 및 수술(양방)`, `이학요법료/이학요법(양방)`, `영상진단 및 방사선치료료/진단(양방)`, `주사료/피하,근육내(한방)`, `특수장비/CT진단(양방)`.
- `코드명` 예: `초진진찰료-의원`, `치석제거[전악]`, `경피적전기신경자극치료[TENS]`, `Cone beam 전산화단층영상진단-일반`, `추나요법-단순추나`, `화상처치-9%이하의범위...`.
- 이 문서가 **수술/시술/물리치료/영상진단/입원** 행위의 유일한 근거다.

## 2. 좌표 기반 파싱 파이프라인 (단계별)

PDF.js로 `page.getTextContent()` 호출 시 각 item은 `{ str, transform }` 형태다. `transform[4]=x`, `transform[5]=y`. y는 아래가 0이라 위로 갈수록 커진다(페이지 좌표). 행 정렬 시 y 내림차순 = 위에서 아래.

### Step 1. 페이지별 텍스트 조각 수집
- 모든 페이지를 순회하며 `{ str, x, y, height }` 조각 배열을 만든다.
- **노이즈 선제거**(정확히 일치 또는 포함으로 필터):
  - 워터마크 조각: `본인`, `외`, `열람금지` 및 그 결합 형태. (워터마크는 보통 큰 폰트·회전·특정 색이라 `transform`의 회전성분이나 폰트 height로도 구분 가능하나, 우선 문자열 기반으로 제거)
  - 헤더 문구: `순번`, `진료시작일`, `병·의원&약국`, `진단과`, `입원/`, `외래`, `주상병`, `코드`, `주상병명`, `내원`, `일수`, `총 진료비`, `처방/`, `조제`, `약품명`, `성분명`, `1회`, `투약량`, `1일`, `투여횟수`, `총`, `투약일수`, `진료내역`, `코드명` 등 헤더 토큰. **단, 헤더 토큰은 본문 데이터(예: 코드명 "코드명"이라는 글자가 본문엔 안 나옴)와 충돌하지 않으므로 y좌표가 헤더 band에 있는 것만 지우는 게 더 안전하다.**
  - 상·하단 고정문구: `2026-6-11`로 시작하는 출력시각, `· 본 자료는...`, `· 진료비에는...`, `· 이에 본 자료를...`, `N/M` 페이지번호.
- **권장 구현**: 페이지마다 헤더 band의 y 하한선을 한 번 찾고(헤더 마지막 줄 y), 그 위쪽 + 하단 안내문구 y 아래쪽 조각을 통째로 버린 뒤 본문 band만 남긴다. 이게 토큰 일일이 매칭하는 것보다 견고하다.

### Step 2. 논리적 행(row) 묶기 — 순번 앵커 방식
- **각 데이터 행은 "순번"(정수, 1열 최좌측 x)으로 시작한다.** 순번 조각의 y를 그 행의 기준 y로 삼는다.
- 한 행은 보통 여러 줄(멀티라인 셀 때문)에 걸친다. **다음 순번이 등장하기 전까지의 모든 조각 = 같은 행**으로 묶는다. (단순 y±오차 묶기보다 "순번~다음순번 사이 y범위"로 묶는 게 멀티라인 셀에 강하다.)
- 순번 인식: 본문 band 최좌측 컬럼 x 근처(±오차)에서 등장하는 순수 정수 토큰. 페이지가 바뀌어도 순번은 누적 증가하므로, 행을 (페이지, 순번)으로 식별.

### Step 3. 컬럼 분해 — 고정 x경계 + 패턴 앵커 병행
이 양식은 컬럼 x좌표가 고정이므로 **컬럼 경계 x값을 상수로 박아도 된다.** 다만 좌표가 미세하게 흔들릴 수 있으니, 신뢰도 높은 필드는 **패턴 앵커**로 잡아 좌표 의존을 줄인다:
- `진료시작일`: `/\d{4}-\d{2}-\d{2}/` — 행에서 이 패턴을 가진 조각이 곧 날짜. (행의 강한 기준점)
- `주상병코드`(기본진료정보): `/^[A-Z]{1,2}\d{2,5}$/` (예: `AR430`, `AM7796`, `AJ0190`, `BM5457`). `$`만 있으면 코드 없음.
- `양/한방`: `/\((양|한)방\)/` — 추출 후 본문에서 분리.
- 금액: `/^[\d,]+$/`.
- 나머지(병의원명, 주상병명, 약품명, 성분명, 진료내역, 코드명)는 **해당 컬럼 x범위 안의 조각들을 y→x 순으로 정렬해 연결**한다.

### Step 4. 멀티라인 셀 병합 (가장 중요)
실제 데이터에서 한 셀이 줄바꿈으로 쪼개진다. 예:
```
"호매실퍼스트정형외과의" + "원"      → "호매실퍼스트정형외과의원"
"(양방)상" + "세불명의" + "골부착부" + "병증, 아래" + "다리"  → "(양방)상세불명의 골부착부병증, 아래다리"
"휴로펜정(록소프로펜" + "나트륨수화물)_(" + "68.1mg/1정)"   → "휴로펜정(록소프로펜나트륨수화물)_(68.1mg/1정)"
```
- 병합 규칙: **같은 행(Step2) 안에서, 같은 컬럼 x범위에 속하는 조각들을 y 내림차순(위→아래)으로 이어붙인다.**
- **공백 처리**: 한국어 의약품/상병명은 줄바꿈 지점에 공백이 없을 수도, 있을 수도 있다. 원문 셀은 시각적 줄바꿈일 뿐이므로 **기본은 공백 없이 연결**하되, 조각 끝/시작이 한글-한글이면 붙이고, 토큰이 명백히 단어 경계(쉼표 뒤 등)면 원문 보존. 과도하게 정규화하지 말 것 — 상병명/약품명은 **후속 매핑 사전의 키**로 쓰므로 원문 형태를 최대한 보존해야 사전 매칭이 된다.
- 단위 토큰(`mg`, `정`, `캡슐`, `mL`, `g`)이 줄 끝에서 쪼개지는 경우가 많으니 약품명 컬럼은 특히 주의.

### Step 5. 행 객체로 정규화
각 문서 타입별 TypeScript 인터페이스로 변환:
```ts
interface BaseRow {           // 기본진료정보
  seq: number;
  date: string;               // YYYY-MM-DD
  facility: string;           // 병·의원&약국 (집계 후 출력에서는 제거)
  dept: string;               // 진단과
  visitType: '외래' | '입원';
  dxCode: string | null;      // 주상병코드, '$'/없음이면 null
  dxType: '양방' | '한방' | null;
  dxName: string;             // 주상병명 (양/한방 표기 제거한 순수명)
  // 금액필드는 옵션
}
interface RxRow {             // 처방조제정보
  seq: number; date: string; facility: string;
  rxType: string;             // 처방/조제
  drugName: string;
  ingredient: string;
  daysTotal: number;          // 총투약일수
}
interface DetailRow {         // 세부진료정보
  seq: number; date: string; facility: string;
  category: string;           // 진료내역 (예: 처치 및 수술/처치 및 수술(양방))
  codeName: string;           // 코드명 (시술/검사명)
}
```

## 3. 함정 목록 (반드시 피할 것)

1. **워터마크 오염**: "본인 외 열람금지"가 본문 조각에 섞임 → 상병명·약품명 중간에 끼어 매핑 실패. Step1에서 반드시 제거.
2. **멀티라인 셀**: 단순 텍스트 순서 추출은 컬럼이 뒤섞임. 좌표 없이는 절대 안 됨. → PDF.js 좌표 필수, 범용 텍스트 추출(`pdf-parse` 등) 금지.
3. **남이 만든 표 추출기 (`pdf-table-extractor` 류) 사용 금지**: 이 PDF의 줄바꿈 깨짐과 워터마크를 못 잡는다. 고정 양식이므로 직접 좌표 재구성이 더 정확하고 통제 가능하다.
4. **양/한방 표기 위치**: 코드 칸이 아니라 상병명 앞에 붙는 경우가 있음. 코드 그룹핑·출력에서 양/한방을 구분해야 하므로 분리 추출 필수.
5. **약국 조제행의 `$`/"해당없음"**: 상병코드 없는 행. null 처리하고 통원 횟수 카운트에서 제외(약국행은 진찰이 아님).
6. **페이지 경계 행**: 행이 페이지 끝에서 잘리지 않는지 확인. 순번 연속성(1,2,3...)으로 누락/중복 검증.
7. **숫자 0 셀**: 세부/처방에서 `1회투약량`이 `0`인 행 존재(점안액 등). 정상값이므로 버리지 말 것.

## 4. 검증 (파서가 맞게 동작하는지 자가 점검)

구현 후 아래로 검증한다 (제공된 3개 샘플 PDF 기준 기대치):
- **기본진료정보**: 순번 1~92, 총 92행. 마지막 행 seq=92, date=2021-06-26, dxCode=null($), dxName=해당없음.
- **처방조제정보**: 순번 1~347, 총 347행.
- **세부진료정보**: 순번 1~338, 총 338행.
- 순번이 1부터 끝까지 **빠짐/중복 없이 연속**이면 행 분리가 정상.
- 스폿체크: 기본진료정보 seq=15 → `2025-04-02 / AK0530 / (양방)만성 단순치주염`. 세부진료정보에 `치석제거[전악]`, `Cone beam 전산화단층영상진단-일반`이 존재. 처방조제정보 seq=4 → 약품명에 `휴로펜정`이 끊김 없이 복원되는지.
- 워터마크 문자열이 어떤 필드에도 남아있지 않은지 grep.

## 5. 모듈 경계

- `parsePdf(file, docType): Promise<BaseRow[] | RxRow[] | DetailRow[]>` 형태로 노출.
- 파서는 **join·집계·시술명 변환을 하지 않는다.** 깨끗한 행 배열까지만 책임진다.
- 집계 모듈(별도)이 세 배열을 `(date + facility)`로 묶고, 상병코드 그룹핑(앞자리 매칭)·통원횟수·투약일 합산·시술명 사전 매핑·고지의무 정리표 생성을 담당한다.
